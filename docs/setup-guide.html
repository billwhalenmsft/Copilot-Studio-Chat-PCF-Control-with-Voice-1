Step 1

Prerequisites

Ensure you have the following before starting:

✓ Azure subscription (Contributor access)
✓ Dynamics 365 / Power Platform environment
✓ Microsoft Copilot Studio bot with Web channel
✓ Power Platform admin permissions
💡
Copilot Studio Direct Line Secret: In Copilot Studio, go to Settings → Channels → Web → Channel Security and copy the Direct Line Secret. You'll need this in Step 4.
📖 Learn more about Web channel security →

Step 2

Deploy Azure Resources

Click the button below to automatically provision Azure Speech Services and Azure OpenAI in your subscription.

Opens the Azure Portal with a pre-configured ARM template

What Gets Created

Resource Type Purpose
{prefix}-speech 🎙️ Speech Services Premium neural voice TTS & speech-to-text
{prefix}-openai 🤖 Azure OpenAI Natural OpenAI TTS voices & chat capabilities
gpt-4o-mini 🧠 Model Deployment Cost-effective model for TTS and general use

Parameters to Fill In

Resource Prefix
A unique name prefix (e.g., contoso). Creates contoso-speech and contoso-openai.
Location
Azure region for Speech Services.
Default: resource group location
OpenAI Location
Azure region for OpenAI. Must support Azure OpenAI.
Default: eastus2
Speech Service SKU
F0 = Free (limited) · S0 = Standard (pay-as-you-go)
Default: S0
  1. Click Deploy to Azure and sign in to your subscription
  2. Select or create a Resource Group
  3. Enter a unique Resource Prefix (lowercase, 3-15 characters)
  4. Choose your regions and SKU
  5. Click Review + Create, then Create
  6. Wait for deployment to complete (~2-3 minutes)
📋
Step 3

Copy Output Values

After deployment completes, go to the Outputs tab in the Azure Portal and copy each value.

🎯
In the Azure Portal: Deployments → your deployment → Outputs. Each value maps to a Dynamics 365 environment variable.
Azure Output → D365 Environment Variable Description
speechKey bw_SpeechKey Azure Speech subscription key
speechRegion bw_SpeechRegion Region code (e.g., eastus)
openAIEndpoint bw_OpenAIEndpoint Full OpenAI endpoint URL
openAIKey bw_OpenAIKey Azure OpenAI API key
openAIDeploymentName bw_OpenAIDeployment Model deployment name
🔐
Security: These keys are sensitive. Store them securely and avoid sharing them in plain text. Consider linking environment variables to Azure Key Vault for production.
📦
Step 4

Import the Solution

  1. Download the solution file: D365MobileCanvasTemplate.zip
  2. Open the Power Apps Maker Portal
  3. Select your target environment from the top-right picker
  4. Go to Solutions in the left navigation
  5. Click Import solution → Browse → select the zip file
  6. On the Environment Variables page, enter values from Step 3:
    • bw_DirectLineSecret — Your Copilot Studio Direct Line secret (from Step 1)
    • bw_SpeechKey — From Azure output speechKey
    • bw_SpeechRegion — From Azure output speechRegion
    • bw_OpenAIEndpoint — From Azure output openAIEndpoint
    • bw_OpenAIKey — From Azure output openAIKey
    • bw_OpenAIDeployment — From Azure output openAIDeploymentName
  7. Set up Connection References when prompted (Dataverse, Outlook, etc.)
  8. Click Import and wait for completion
⏱️
Solution import typically takes 3-5 minutes. You'll receive a notification when complete.
🎉
Step 5

Configure & Launch the App

  1. In the Maker Portal, go to Apps
  2. Find the D365 Mobile Canvas Template app
  3. Click Edit to open in Power Apps Studio
  4. Add the Environment Variable Values data source:
    • Click Data in the left panel → Add data
    • Search for "Environment Variable Values"
    • Select it to add as a Dataverse data source
  5. Verify the chat control loads and connects to your Copilot Studio bot
  6. Click Save → Publish
  7. Share the app with your users via Share
✅
You're all set! The app should now display the Copilot Chat control with voice input/output, adaptive cards, and driving mode capabilities.
Optional Features

Optional

Camera & Vision AI

Enable the camera button for equipment inspection, barcode scanning, or visual analysis powered by Azure OpenAI GPT-4o Vision.

Environment Variable Value
bw_VisionEndpoint Your Azure OpenAI endpoint (can reuse openAIEndpoint if GPT-4o is deployed there)
bw_VisionApiKey API key for the Vision-capable OpenAI resource
bw_VisionSystemPrompt Custom prompt for image analysis (e.g., "You are an equipment inspector…")
💡
The camera button only appears when EnableCamera is set to Yes on the PCF control and bw_VisionEndpoint + bw_VisionApiKey are configured.
Troubleshooting
🔧

Common Issues

Chat control shows "Unable to connect"
  • Verify bw_DirectLineSecret is set correctly
  • Ensure the Copilot Studio bot Web channel is enabled
  • Check that bw_DirectLineEndpoint is https://directline.botframework.com/v3/directline
Voice not working
  • Confirm bw_SpeechKey and bw_SpeechRegion are set
  • Ensure the app is served over HTTPS (required for microphone access)
  • Check browser microphone permissions
  • Try using Admin/Debug Mode to view detailed logs
"Name isn't valid" error in Canvas App
  • You need to add the Environment Variable Values data source
  • In Power Apps Studio: Data → Add data → search "Environment Variable Values"
  • This is a built-in Dataverse table — no need to add it to the solution
Azure deployment fails
  • Ensure your subscription has the Microsoft.CognitiveServices provider registered
  • Check that your chosen region supports Azure OpenAI (not all regions do)
  • Verify you have Contributor role on the resource group
  • If you hit quota limits, try a different region or request a quota increase
